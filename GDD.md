# BJCP 精酿啤酒酿造师 - 游戏设计文档 (GDD)

## 1. 游戏概述

**游戏名称**：精酿啤酒酿造师 (Craft Beer Brewmaster)  
**平台**：手机端网页 + 桌面端网页（响应式 HTML5）  
**核心体验**：让玩家体验从选风格 → 配原料 → 控温度 → 成品评分的完整酿酒过程  
**目标受众**：精酿啤酒爱好者、BJCP学习者、休闲模拟游戏玩家  
**单局时长**：5-10 分钟  
**传播点**：成品分享图（酿酒师手持啤酒赞不绝口）

---

## 2. 核心玩法循环

```
选择风格 → 选择酿酒师 → 糖化(麦芽+温度) → 煮沸(酒花+时间) 
→ 发酵(酵母+温度+时间) → 二次/增味(可选) → 成品评分 → 命名分享
```

## 5. 酿造流程详解

### 5.1 糖化 (Mashing)
**目标**：将麦芽淀粉转化为可发酵糖

**玩家操作**：
- 选择基础麦芽（2-6种可选，不同风格预设不同选项）
  - 皮尔森麦芽、维也纳麦芽、慕尼黑麦芽、焦糖麦芽、巧克力麦芽、黑麦芽、小麦麦芽、燕麦
- 设置糖化温度：60-72°C
- 设置糖化时间：30-90分钟

**酿酒师反馈**：
- 温度 < 62°C："温度偏低，酶活性不足，可能导致发酵度不够。"
- 温度 > 70°C："温度太高！酶会失活，糖化不完全！"
- 时间 < 45分钟："时间太短，糖化可能不充分。"
- 深色麦芽过多（对淡色拉格）："这么多深色麦芽？这款酒颜色会太深了！"

**影响参数**：OG、SRM、发酵度

---

### 5.2 煮沸 (Boiling)
**目标**：杀菌、浓缩、提取酒花苦味和香气

**玩家操作**：
- 选择酒花品种（3-8种可选）
  - 苦花：马格努门、世纪、哥伦布
  - 香花：卡斯卡特、西楚、马赛克、银河
  - 传统花：萨兹、哈拉道、泰南格
- 设置酒花投放阶段和数量：
  - 苦花：煮沸开始（60分钟）
  - 风味花：煮沸中（15-30分钟）
  - 香花：煮沸结束（0-5分钟）
  - 干投：发酵后（不计入煮沸）
- 设置煮沸时间：60-120分钟

**酿酒师反馈**：
- IBU 超出风格上限："酒花太多了！这款风格不需要这么苦！"
- IBU 低于风格下限："酒花不够，苦味支撑不住麦芽甜味。"
- 香花投放过早："香花煮太久会损失香气，建议最后5分钟投放！"
- 干投量过大："干投太多可能带来青草味和涩感。"

**影响参数**：IBU、OG（浓缩）、风味特征

---

### 5.3 发酵 (Fermentation)
**目标**：酵母将糖转化为酒精和CO₂

**玩家操作**：
- 选择酵母菌株：
  - 拉格酵母（下层发酵，4-12°C）
  - 艾尔酵母（上层发酵，15-22°C）
  - 小麦啤酒酵母（产酚类，17-20°C）
  - 比利时酵母（产酯类，18-24°C）
  - 布雷特酵母（野菌，20-25°C）
  - 乳酸菌（酸化，20-30°C）
- 设置发酵温度
- 设置发酵时间：7-30天

**酿酒师反馈**：
- 拉格风格选艾尔酵母："这款风格应该用拉格酵母，艾尔酵母会带来太多果香！"
- 拉格温度 > 15°C："拉格必须在低温发酵！高温会产生杂醇和酯类！"
- 艾尔温度 < 12°C："温度太低，艾尔酵母会休眠，发酵停滞！"
- 发酵时间太短："发酵还没完成，残糖会太高！"
- 比利时风格温度太低："比利时酵母需要温暖环境才能产生复杂酯香！"

**影响参数**：FG、ABV、风味特征（酯类、酚类）

---

### 5.4 二次发酵 / 增味 (Conditioning & Flavoring)
**可选步骤**，根据风格决定是否开放

**玩家操作**：
- 添加水果（樱桃、覆盆子、桃子、百香果、橙子…）
- 添加香料（香菜籽、橙皮、八角、香草…）
- 添加野菌（布雷特、片球菌、乳酸菌）
- 木材陈酿（橡木片、波本桶、红酒桶）
- 设置二次发酵时间：1-12个月

**酿酒师反馈**：
- 水果过多："水果味会淹没啤酒本身，建议减少！"
- 香料过多："香料太抢戏了，啤酒不是香料汤！"
- 在传统风格加野菌："这款经典风格通常不使用野菌，你确定要实验吗？"
- 酸啤不加酸菌："这款风格需要酸味支撑，不加乳酸菌怎么酸？"
- 帝国世涛加水果："帝国世涛加水果？这是个大胆的创新！"

**影响参数**：风味复杂度、酸度、特殊风味

## 6. 评分系统

### 6.1 参数匹配评分（60%权重）
将玩家最终参数与 BJCP 目标范围对比：

| 参数 | 完美匹配 | 接近（±10%） | 偏差（±20%） | 严重偏差 |
|------|---------|-------------|-------------|---------|
| OG | 15分 | 12分 | 8分 | 3分 |
| FG | 15分 | 12分 | 8分 | 3分 |
| IBU | 15分 | 12分 | 8分 | 3分 |
| SRM | 15分 | 12分 | 8分 | 3分 |
| ABV | 15分 | 12分 | 8分 | 3分 |

### 6.2 工艺评分（20%权重）
- 温度控制是否合适
- 酵母选择是否匹配风格
- 酒花投放时机是否合理
- 发酵时间是否充足

### 6.3 创意评分（20%权重）
- 在风格允许范围内创新加分
- 严重偏离风格扣分
- 特种原料使用是否和谐

### 6.4 评级
| 总分 | 评级 | 酿酒师评语 |
|------|------|-----------|
| 95-100 | 🏆 大师级 | "完美！这可以去参加比赛了！" |
| 85-94 | ⭐ 优秀 | "非常出色，接近商业水准！" |
| 70-84 | 👍 良好 | "不错的尝试，还有提升空间。" |
| 50-69 | 🤔 一般 | "能喝，但离目标风格还有距离。" |
| <50 | 😅 实验品 | "这是一次…有趣的实验。" |

## 7. 成品分享系统

### 7.1 生成图片内容
使用 HTML Canvas 生成 1080×1920（手机竖屏）分享图：

**画面布局**：
- 顶部：酒名（玩家输入，大字）
- 中上：风格名称 + 评级徽章
- 中间：啤酒杯插图（颜色根据 SRM 动态变化）
- 中右：5项参数雷达图或条形图
- 中下：酿酒师形象（手持啤酒杯，竖起大拇指）
- 底部：酿酒师评语 + 游戏Logo + 二维码

**啤酒颜色算法**：
SRM 2-3 → 淡稻草色  
SRM 4-6 → 金色  
SRM 7-14 → 琥珀色  
SRM 15-25 → 棕色  
SRM 26-40 → 深棕/黑色  

### 7.2 分享方式
- 长按保存图片
- 一键复制分享文案

## 9. 技术架构

### 9.1 技术栈
- **前端**：纯 HTML5 + CSS3 + JavaScript（ES6+）

### 9.3 酿造计算引擎（简化模型）
```javascript
// 核心公式（简化版，用于游戏）
OG = f(麦芽种类, 麦芽用量, 糖化效率)
FG = f(OG, 酵母 attenuation, 发酵温度, 发酵时间)
ABV = (OG - FG) * 131.25
IBU = Σ(酒花α酸% × 酒花克数 × 利用率) / 体积
SRM = Σ(麦芽色度 × 麦芽磅数) / 体积
```
